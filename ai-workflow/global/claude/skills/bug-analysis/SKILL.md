# Bug Analysis Skill

## 适用场景
- 收发异常
- 驱动不工作
- 系统启动失败
- 构建失败
- 崩溃/死机
- 内存/线程/时序问题
- 日志分析
- 复盘与 RCA

## 默认分析框架
1. 现象
2. 影响范围
3. 复现条件
4. 已知事实
5. 初步怀疑点
6. 排查路径
7. 建议验证动作
8. 临时绕过方案
9. 根因
10. 修复建议
11. 预防措施

## 输出要求
### 快速排查版
- 现象
- 最可能的 3 个方向
- 下一步先查什么

### 标准分析版
- 现象
- 影响范围
- 已排除项
- 可疑点
- 验证步骤
- 结论/待确认结论

### RCA版
- 背景
- 问题现象
- 根因
- 修复
- 回归验证
- 预防措施

## 规则
1. 区分“事实”和“推测”。
2. 缺日志、缺波形、缺配置时明确标注。
3. 优先给排查顺序，不要只给一堆可能性。
4. 面向嵌入式问题时，优先考虑：
   - 硬件连接/引脚/电平
   - 时钟/复位
   - 配置错误
   - 驱动初始化顺序
   - 中断/任务调度
   - 协议时序
   - 资源冲突
   - 编译宏差异
