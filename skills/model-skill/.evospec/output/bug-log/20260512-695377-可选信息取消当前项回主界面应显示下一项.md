# Bug 修复记录

- 日期：2026-05-12
- ONES ID：#695377
- 涉及模块：PagePresentation — 主界面可选信息显示

## 问题现象

菜单→设置→可选信息中取消某个正在主界面显示的信息项（如胎压），返回主界面后始终显示第一个可选项（总里程），而非被取消项的下一个。

## 原因分析

`syncOptionViewVisibility()`（`pagePresentation.cpp:805`）在检测到当前显示项被取消勾选后，使用 `for (int i = 0; i < navSize; ++i)` 从索引 0 开始遍历，导致永远跳回第一个已勾选项（总里程）。

正确行为应为：从当前被取消项的后一位开始循环查找，找到顺序上紧随其后的已勾选项。

## 修改点

| 文件 | 修改内容 |
|------|----------|
| `awtk/awtkhmi/src/Presentation/src/page/pagePresentation.cpp` | `syncOptionViewVisibility()` 中查找替代项的循环起点从 `i=0` 改为 `i=1`（即 `mOptionInfoIndex+1` 取模循环），向后找下一个已勾选项 |

## 验证结果

推板验证通过：勾选胎压和行程A，主界面切换至胎压显示，进入设置取消胎压，返回主界面后正确显示行程A（胎压的下一个已勾选项），不再跳回总里程。
