# Bug 修复记录：时间设置分钟确认后小时变 00

- 日期：2026-05-13
- ONES ID：未提供
- 模块：awtkhmi 设置 / 时间设置
- 日志来源：`.evospec\bugLog\[com COM5]  (2026-05-13_100243)  COM4  (USB Serial Port (COM4)).log`

## 问题现象

菜单 -> 设置 -> 时间设置界面，时间格式为 24 小时，小时为 15、分钟为 28。选中分钟后调到 30 并确认，偶发出现小时设定值从 15 变成 00。

## 原因分析

`SettingData::setTimeMinute()` 组包时使用内部缓存 `mTimeHour` 作为下发小时值。该缓存默认值为 0，若用户只修改分钟且当前界面小时值来自 MCU 回显缓存，`mTimeHour` 可能未被同步为真实小时，导致 HMI 下发 `Hour=0, Minute=30`。

问题日志中可见分钟设置下发包携带 `Hour=0, Minute=30`，随后 MCU 回显 `Hour=0, Minute=30`，界面显示更新为 00:30。

## 修改内容

- 设置小时前，从 MCU 回显缓存同步当前分钟，避免只修改小时时携带默认或陈旧分钟。
- 设置分钟前，从 MCU 回显缓存同步当前小时，避免只修改分钟时携带默认或陈旧小时。
- 对回显小时、分钟做协议范围检查，异常时保留原缓存值并打印告警日志。
