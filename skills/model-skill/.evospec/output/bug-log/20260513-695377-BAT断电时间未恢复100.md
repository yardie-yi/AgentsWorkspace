# Bug 修复记录

- 日期：2026-05-13
- ONES ID：#695377
- 涉及模块：BasicPresentation（时间显示逻辑）

## 问题现象

断开 B+ 与 IG 电源后，重新上电，仪表时间显示未恢复为初始值 1:00。
需求：BAT+IGN 同时 OFF（电池更换）时钟应恢复 1:00；BAT 掉电后首次 IGN ON 时钟区块 1Hz 闪烁 10s 提示重新设定。

## 原因分析

**Bug 1 — 时间未恢复 1:00**
BAT 掉电后 MCU RTC 丢失，通过 MCU2CLU_8301 Data11 下发 timeType=0（RTC 无效）。
basicPresentation.cpp:updateTime() 的 else 分支直接 return，控件从未被写入"01:00"。

**Bug 2 — 闪烁计数可能不完整**
updateRtcState() 启动 SELF_TIMER_ID29 前未重置 mFlashRtcCount，导致 10s 闪烁提前结束。

## 修改点

| 文件 | 修改内容 |
|------|----------|
| Presentation/src/basic/basicPresentation.cpp | updateTime() else 分支：缓存非1:00时写控件"01:00"并更新缓存，再 return |
| Presentation/src/basic/basicPresentation.cpp | updateRtcState()：启动定时器前先将 mFlashRtcCount=0 |

## 验证要点

1. 断 B+/IG 后重新上电，三处时间控件均显示"01:00"
2. 日志确认：RTC无效(timeType=0)，显示初始时间01:00
3. MCU2CLU_840B rtcStartData 非零时，1Hz 闪烁恰好 10s 后停止
4. 用户设定时间后显示正常更新、不再闪烁
5. 正常 IGN off/on（不断 B+）：行为不变
