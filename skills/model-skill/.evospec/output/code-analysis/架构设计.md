## 1. 项目概述

### 1.1 项目简介

AWTK HMI 是一个基于 AWTK 框架开发的汽车仪表盘人机交互界面应用程序。项目采用标准的 MVC（Model-View-Controller）架构模式，参照 Kanzi Cluster 项目的架构设计，实现了清晰的职责分离、模块化管理和高可维护性。

### 1.2 基本信息

| 属性 | 值 |
|------|-----|
| 项目名称 | awtkhmi |
| 框架 | AWTK (A Toolkit for Embedded GUI) |
| 编程语言 | C/C++17 |
| 架构模式 | MVC (Model-View-Controller) + Presentation Layer |
| 参考项目 | Kanzi Cluster HMI |
| 屏幕分辨率 | 800x480 |
| 颜色深度 | 16位 |
| 支持语言 | 中文/英文 |
| 目标平台 | ARM Linux (交叉编译) |
| 构建系统 | SCons + CMake |

### 1.3 支持车型

项目支持多种车型配置：

| 车型代码 | 车型名称 |
|---------|---------|
| enum_P701_VEHICLE_TYPE | P701 车型 |
| enum_SC2E_VEHICLE_TYPE | SC2E 车型 |
| enum_SC26_VEHICLE_TYPE | SC26 车型（默认） |

### 3.3 数据流向（Pull Model）

项目采用 **Pull Model（主动拉取模型）**，Presentation 层主动从 Model 层拉取数据：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         Pull Model 数据流                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   1. 数据接收流程 (Linux 环境)                                       │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │                                                              │ │
│   │   Vehicle Interface (CAN/串口/蓝牙/WiFi)                    │ │
│   │         │                                                    │ │
│   │         ▼                                                    │ │
│   │   DataDispatcher::notify()  ── 接收消息                      │ │
│   │         │                                                    │ │
│   │         ▼                                                    │ │
│   │   DataDispatcher 存储数据到内部变量                          │ │
│   │   (mGaugeData, mTelltalesStates, etc.)                     │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   2. 数据分发流程                                                   │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │                                                              │ │
│   │   DataDispatcher::dispatch()                                │ │
│   │         │                                                    │ │
│   │         ▼                                                    │ │
│   │   遍历 objVector (所有 ModelData 实例)                        │ │
│   │         │                                                    │ │
│   │         ▼                                                    │ │
│   │   ModelData::onData() ── 各子模块处理数据                     │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   3. 数据拉取流程 (Pull Model)                                      │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │                                                              │ │
│   │   Presentation::onUpdate() (每帧调用)                        │ │
│   │         │                                                    │ │
│   │         ▼                                                    │ │
│   │   调用 DataDispatcher::getXxxData() 获取数据                 │ │
│   │         │                                                    │ │
│   │         ▼                                                    │ │
│   │   更新 UI 控件 (widget_set_prop_xxx)                         │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   4. 数据接口示例                                                   │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │                                                              │ │
│   │   // 获取仪表数据                                            │ │

│   │   // 获取指示灯状态                                          │ │

│   │   // 获取警告数据                                            │ │
│   │   std::vector<sWarningData> getWarningData() const;         │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 13. 扩展指南

### 13.1 添加新模块

1. **创建 Model 层**
   - 继承 `ModelData` 基类
   - 实现 `init()`, `deinit()`, `onData()` 方法
   - 在 `DataDispatcher` 中添加数据存储

2. **创建 Controller 层**
   - 继承 `Controller` 基类
   - 实现 `init()`, `deinit()`, `onControl()` 方法
   - 如需处理按键，重写 `onKeyManagement()`

3. **创建 Presentation 层**
   - 继承 `Presentation` 基类
   - 实现 `init()`, `deinit()`, `onUpdate()` 方法
   - 从 DataDispatcher 拉取数据更新 UI

4. **注册模块**
   - 在 `commonProj.h` 中添加对象枚举
   - 在 `AwtkHmi` 类中添加成员变量
   - 在 `registerInstance()` 中实例化

### 13.2 添加新页面

1. 在 AWTK Designer 中设计 UI
2. 生成 XML 文件到 `design/default/ui/`
3. 创建对应的 Page 文件到 `src/pages/`
4. 在 `navigator` 中注册页面
