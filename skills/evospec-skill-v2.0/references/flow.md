## 三、执行流程

```
1. Read .evospec/memory/project_state.yaml
2. 确定当前Phase和Step
3. Read对应phase详细文档
   - references/phases/{phase}.md
4. 执行阶段任务
5. Update project_state.yaml
6. 继续下一Step
```

---

## 四、并行执行

可并行的Steps：

| 时机 | 可并行Steps |
|------|------------|
| S2.2完成后 | [S3.1, S3.2] |
| S3.3完成后 | [S3.4, S3.5] |
| S4.2完成后 | [S4.3, S4.4] |

执行方式：
```
检查依赖就绪 → 同时执行 → 等待全部完成 → 继续
```

---

## 五、状态恢复

```
Read project_state.yaml:
├── 存在未完成Step → AskUserQuestion:
│   "上次停在{Step}，是否继续？"
│   [继续] [从某Step重新开始] [重新开始]
└── 无状态或已完成 → 从初始化开始
```

---

## 六、规则加载

每个阶段执行前：

```
1. Read .evospec/config.yaml
2. Read .evospec/rules-global.yaml
3. 过滤: status=enabled AND phase包含当前阶段
4. 匹配: trigger_pattern匹配上下文
5. 注入: 所有匹配规则
```

## 七、与敏捷模式切换

### 流程化 → 敏捷

```
用户: "快速修复XX Bug"
→ 切换到敏捷模式
→ 规则库共享
→ change_log记录变更
```

### 敏捷 → 流程化

```
用户: "开始正式开发XX模块"
→ 从S1.1开始（或跳过已完成阶段）
→ 继承已有规则库
```
