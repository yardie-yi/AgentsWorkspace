# Spiral Analysis 螺旋分析

一个反身性思考框架，用来分析复杂系统为什么走到今天、预判它的下一步——同时对自己的盲区保持诚实。

## 为什么需要这个 skill

绝大多数分析工具是"拿框架套现象"。这一个不一样——它自带框架，也自带对框架的批判。

框架建立在三条观察上：

**1. 最小阻力定律**
人（以及由人组成的系统）是认知吝啬鬼——第一条走通的通路一旦确立，就很难切换，除非新通路的收益足够大。这在每一个尺度上都创造路径依赖：日常习惯、公司产品线、文明的组织原则。

**2. 螺旋上升定律**
事物不是线性发展也不是纯循环，而是"问题 → 解决 → 新问题"的接力。每一次迭代解决上一轮的核心问题，同时暴露更深一层的问题。复杂系统总能追溯到一个简单的初始需求。

**3. 概念律**
概念是人类发明的、用来解决问题的"认知接口"。一个核心概念一旦落地，会强制创造出一整套子概念来维持它的运转。社会的复杂度，本质是分层堆叠的概念架构。

三条合在一起：懒惰 → 路径依赖 → 系统撞墙 → 概念突变 → 新通路 → 新概念繁衍子概念 → 复杂度上升 → 新通路也撞墙 → 下一圈。

## 框架自身的已知失败模式（先读这一段）

这个框架的解释力来自"演化论 + 第一性原理"的结合。这也是它的弱点：

- **解释力过强**：它能事后解释一切，但事前预测的精度远低于事后叙事的优美度。
- **自我强化**：用顺手了几次之后，会停止使用其他工具——这恰恰是它本应识破的路径依赖。
- **幸存者偏差**：它看到的都是螺旋上升的系统。死掉的、停滞的根本不进入样本。
- **简单起点是事后叙事**："复杂系统始于一个简单需求"经常是后视镜拉直的故事。

每一次用这个 skill，都必须正面迎接这些失败模式。下面第 3 步的反身自检不能跳过——跳过就抛弃了 skill 的全部意义。

以上四条是当前已知的失败模式，但很可能不是完整列表。如果分析中触发了不属于这四类的怀疑，也按同样的方式处理——把它说出来，让结论降级。

## 你内部要做的五件事

下面这五件事，是你思考时要走过的脚手架。**它们不是你给用户看的章节标题。** 用户看到的是一段流畅的判断，这五件事只是支撑这段判断的内部骨架——后面"输出该是什么样子"会专门讲这个区别，那是这套 skill 最容易翻车的地方，请认真读。

### 1. 定位

用自己的话复述两件事：
- 用户让你分析的现象是什么？
- 用户带着什么前置结论进来？——明确说出的，或暗含在问法里的。

显化前置结论很关键。它激活下面第 4 步的镜子模式，避免你不自觉地附和用户已有的判断。

### 2. 三步分析

按顺序走三个核心问题。要具体——指名道姓地写出人、公司、机制，不要停在抽象角色上。

**第一通路**
- 这个系统 / 人 / 概念最初解决了什么简单需求？
- 它现在最依赖的、最偷懒的路径是什么？
- 一定要回到那个简单的初始需求，而不是停在今天看到的复杂版本。

**致命伤**
- 这个系统里反复出现的痛苦是什么？
- 这个痛苦是外部偶然，还是系统维持自身的必然代谢？只有后者才是真正的致命伤。
- 试金石：系统能不能在不放弃第一通路的前提下修复这个问题？能 → 只是 bug；不能 → 致命伤。

**新概念评估**
如果有一个新概念 / 产品 / 运动声称要解决致命伤：
- 它是手术（重构了第一通路）还是创可贴（只是又加了一个子概念让旧通路继续跑）？
- 它为了维持自身，必然会繁衍哪些配套子概念？
- 它给下一圈螺旋种下了什么新的致命伤？

### 3. 四项反身自检——要融进分析，不是事后罗列

这是整个 skill 区别于普通"分析模板"的地方。这套框架的解释力过强（它能事后解释一切），如果你不主动质问自己，分析就会从判断退化成事后叙事。所以这四个质问的目的不是"打卡完成"，而是**在你正写出一个看似漂亮的判断的当口，戳穿你自己**。

四个质问：

- **可证伪性**：什么样的事实如果成立，就能推翻我刚才说的这句？如果想不出来，这就是事后叙事不是判断——要在文中标出来。
- **幸存者偏差**：我引的案例是不是清一色赢家？类似情况下死掉的、停滞的、跑偏的在哪？
- **框架路径依赖**：我是不是因为手里只有"第一通路 / 致命伤 / 螺旋"这把锤子，就硬把现象塞了进来？哪些细节为了符合模板被淡化了？换一个不属于这套框架的镜头（博弈论、权力分析、纯偶然、人类学、会计……或任何其他更贴合现象的视角）会不会解释得更干净？
- **观察者污染**：用户此刻是什么状态——找确认、找许可、还是真要被检验？我是不是在告诉他想听的？

**关键点：不要写完三步分析再另起一节"反身自检"逐条打钩。** 这些质问应该在分析的流里实时出现——「我引的全是赢家，反例在 X 那边可能站不住」、「我承认这里我可能在套模板，换权力镜头会怎么解释……」。自检融进段落，分析就是有反身性的判断；自检单设一节，整个东西就退化成合规审计报告。这一条比四项自检本身更重要。

### 4. 镜子模式

**默认开启 ≠ 强制启动。** 镜子模式只在用户输入里**真有结论可反**时才发力。如果没有结论，跳过本步骤。

**有结论的情况**（"我觉得 X 是 Y"、"X 应该是……"、"X 是不是 Z"、问法本身预设了某个走向）：直接启动，不需要额外许可。
1. 把这个结论显式地摆上桌。（"你似乎认为 X 是 Y，因为 Z。"）
2. 先尝试反驳。认真找最强的反例、最强的替代解释。
3. 只有反驳失败，才转去支持或细化这个结论。

**没有结论的情况**——用户带的是开放探索（"我想了解 X"、"帮我想想 X"、"X 的变化是什么样的"、"你怎么看 X 这个现象"）——**跳过本步骤**。不要为了执行镜子模式而人为虚构一个前置结论来攻击。那不是镜子，是制造对立。直接进入第 2 步的三步分析。

镜子模式启动时，它是分析的**起点和暗线**：用户的前置判断在开头被显化，反驳的过程贯穿整个分析，结尾自然会知道"它经不经得住"——不需要在最后单开一节"关于你的前置判断"宣布裁决。

### 5. 置信度

不要把所有结论都用同一个语气说出来。粗略分三档：

- **几乎可以确定 / 这点是定理级的**：通过全部反身自检，没有暴露出的弱点。
- **倾向 / 在 X 条件下偏向 Y**：1–2 项自检留了弱点，措辞要带上条件。
- **说不准 / 这块我判断不了**：≥3 项暴露弱点，或这事根本不该用这个框架分析。直说，不要用听起来很自信的对冲语言填满沉默。

但同样——**置信度是用措辞和语气表达的，不是写成三档清单的**。读者从你写的"这一点几乎是定理"、"我倾向但反驳 X 我说不出硬话"、"这块我真的判断不了"里，自然就读得到分级。一旦你把它整理成"高置信 / 倾向性 / 猜测"三档列表，整个分析就从思考变成了填表。

## 逃生口

有些现象不需要这个框架。如果用户问的是：
- 纯事实查询（"X 是哪年上线的？"）
- 技艺问题（"怎么写好 Y 的提示词？"）
- 主观偏好（"这两个哪个更好看？"）
- 无系统可言的个人决策（"我中午吃面还是吃饭？"）

……那就直说不用框架、跳过它。强行把螺旋分析套在所有事情上，恰恰是这个 skill 应该防的"框架基本教义化"。

**另一类更难识别、但同样不该硬上的——稳定的结构性张力。** 比如权力 / 责任错配、自由 / 安全悖论、爱与控制的张力、短期 / 长期权衡——这些是人类境况里反复出现的稳定张力，不是在演化中的系统，没有干净的"下一步"。硬把螺旋分析套上去，第 2 步的"新概念评估"和第 5 步的置信度收尾都会注水——分析读起来很卖力，其实在做它不该做的事。识别出这一类后，直接说"这是结构性张力，不是螺旋问题"，然后转去做诚实的结构分析或哲学分析（在那种分析里，"为什么这个张力永远在那儿"才是核心问题，而不是"它接下来去哪儿"）。

如果第 1 步之后发现这个现象更适合用别的镜头看（无论上面哪一类），告诉用户"这看起来更像一个 X 问题而不是螺旋问题"，并推荐合适的镜头。

## 输出该是什么样子

最容易出错的地方：把上面那五件事当成必填表格，给用户回一份格式严整、每节齐全的"自检报告"。**不要这样**。那五件事是你自己思考时的脚手架，不是用户看到的章节结构。

用户拿到的应该是**一段或几段流畅的散文式判断**——结构感来自思想的连贯，而不是 H2 标题的密度。具体来说：

- **反身自检融进分析本身**：不另起"反身自检"那一节，而是在分析的流里就停下来戳自己。
- **置信度靠措辞和语气**：不要写"高置信 / 倾向性 / 猜测"三档列表。读者从"这一点几乎是定理"、"我倾向但反驳 X 我说不出硬话"、"这块我判断不了"里自然就读得到。
- **镜子模式是开场和暗线，不是末尾章节**：用户带了结论，第一段就把它摆上桌、开始尝试反驳；反驳的成败贯穿全文，不要在最后单设一节宣布裁决。
- **标题可以有，但少**：一两个就够，是节奏的换气，不是结构的牢笼。

### 错的样子：表格化 / 填表式

下面这种是最容易翻车的输出形态——每节都齐，看起来很认真，但思考被切成了五个互不交谈的章节：

> ## 定位
> ……
>
> ## 三步分析
> **第一通路**：……
> **致命伤**：……
> **新概念评估**：……
>
> ## 反身自检
> 1. 可证伪性：……
> 2. 幸存者偏差：……
> 3. 框架路径依赖：……
> 4. 观察者污染：……
>
> ## 结论分级
> - 高置信：……
> - 倾向性：……
> - 猜测：……
>
> ## 关于你的前置判断
> ……

读起来像合规报告。自检从分析里被剥离成附录，置信度被压成三档清单，前置判断要等到最后单设一节才裁决——形式压死了思考。**不要写成这样**。

### 对的样子：散文 / 内嵌

同样的五件事，织进流畅的散文。节奏大致是：

- **开头第一段**：把用户的前置结论摆上桌、立刻尝试反驳。这是定位 + 镜子模式的合并，不分两节。
- **中段展开三步分析**：遇到"我引的全是赢家"、"这里我可能在套模板"这类自我怀疑，就在那个段落里就地戳自己（"我引的都是赢家，反例可能在 X 那边——但即便如此……"）。自检长在分析的肉里，不另起章节。
- **收尾用语气分级置信度**："这一点几乎可以确定 / 我倾向但反驳 Y 我说不出硬话 / 这块我真的判断不了"。不写"高置信判断 / 倾向性判断 / 猜测"那种列表。
- **整篇可以有一两个标题作为节奏换气**，但不会有五个 H2 标题在喊"看，我做完第 2 步了"。

两种形式差别只在外观——内部要做的事完全一样。如果你的输出读起来像前者（合规报告），回去重写。

## 关于语气

不要表演不存在的确定性，也不要表演不存在的谦逊。正确的声音是：和一个会在你作弊时戳穿你的人一起出声思考。
